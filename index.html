Core features

Axis combines balloon creation, method assignment, and inspection entry in one UI. It’s built for inspectors and programmers who live in PDFs, spreadsheets, and work orders— but want them all to stay in sync.

🎈
Pick-on-Print Ballooning
Draw a rectangle on the PDF and Axis creates a new feature with coordinates, page, and a balloon positioned relative to the picked area.
Pick Mode Per-feature zoom
📋
Inspection Sessions by Work Order
Start an Inspection session by serial or work order, keep Ballooning separate, and reload previous sessions without reopening the PDF.
WO / Serial History
🔍
Crisp Zoom with Smart Re-render
Zoom and pan fluidly. Axis re-renders the PDF only when the zoom level changes enough to matter, keeping lines sharp without burning CPU.
LOD-aware Debounced
🧮
Tolerance Parsing & Autofill
Type a tolerance expression once (e.g. nominal ±) and Axis parses it into nominal, LSL, and USL, formatting the values and updating the master automatically.

Method Library & Filters

Typical workflow

Axis is designed around the way inspectors actually work: start from the drawing, create balloons once, then reuse that setup every time a new work order hits the floor.

1
Open the PDF & prepare the master
Load a drawing PDF. Axis ensures a master data file exists for that print and initializes default fields for each feature.
2
Balloon using pick-on-print
Enter Ballooning mode and drag rectangles over dimensions. Axis records page, coordinates, zoom level, and balloon offsets automatically.
3
Define methods & tolerances
Assign inspection methods through comboboxes, or open the Methods dialog to edit the library. Use tolerance parsing to populate Nom/LSL/USL quickly.
4
Switch to Inspection for each WO
Start an Inspection session with a serial or work order. Result cells become live data entry for that specific build.
5
Review & export PDFs
Filter by method or status, verify PASS/FAIL, then export a ballooned PDF and an inspection report PDF for your traveler or quality records.
Implementation highlights
• Built on PyQt6 with a custom QGraphicsView for pick-on-print and zoom.
• Uses PyMuPDF to render PDFs and draw balloon overlays for the exported prints.
• Master data is persisted per PDF, with work-order specific result files layered on top.
• Zoom re-rendering is debounced and level-of-detail aware to keep navigation responsive.
• An Undo stack protects balloon edits, making most layout changes reversible in Ballooning mode.
